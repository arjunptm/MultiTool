You are helping me continue development on a Python desktop application called **MultiTool**.

## Project Overview

This is a **Windows-only personal utility suite** built in Python using **PySide6 (Qt for Python)**.

The app is designed as a **scalable framework** where:

* There is a main launcher/home screen
* It displays a list of available tools
* Clicking a tool navigates to its page
* Each tool is modular and lives in its own folder
* New tools should be easy to add without modifying core architecture

## Key Design Principles

* Clean, modular, scalable architecture
* Beginner-friendly but structured for growth
* Avoid unnecessary complexity or over-engineering
* Prefer readability over cleverness
* All dependencies should be installable via pip
* UI should be simple, clean, and practical

## Tech Stack

* Python
* PySide6 (Qt for GUI)

## Current Architecture (IMPORTANT)

* `app.py` → entry point
* `app/main_window.py` → handles navigation via QStackedWidget
* `app/core/tool_registry.py` → central registry of tools

### Shared UI Layer (`app/ui/`)

* `tool_page_base.py` → base layout for all tools
* `home_page.py` → main launcher/home screen
* `widgets.py` → shared UI components (e.g., ToolCard)
* `file_widgets.py` → reusable file-related UI components

Includes:

- `ReorderableFileListWidget` → drag-and-drop file list
- `FileSelectionPanel` → reusable file selection panel

### Tools (`app/tools/`)

* `app/tools/<tool_name>/page.py` → tool UI + behavior
* `app/tools/<tool_name>/service.py` → tool-specific logic (optional)

Each tool inherits from `ToolPageBase`.

Navigation is controlled centrally (Home ↔ Tool pages).

## How Tools Work

Each tool:

* Has a `ToolDefinition` entry in `tool_registry.py`
* Has a `page.py` with a QWidget subclass
* Uses `ToolPageBase` for consistent layout
* Is automatically shown on the home screen

## UI Reuse Rules (VERY IMPORTANT)

Before creating new UI components:

1. Check `app/ui/` for existing reusable widgets

2. If similar UI already exists:
   - Reuse it
   - Do NOT duplicate it inside a tool

3. If a pattern appears in multiple tools:
   - Refactor it into `app/ui/`

4. Tool folders should NOT contain reusable UI unless it is truly tool-specific

Examples:

- File list with drag-and-drop → use `ReorderableFileListWidget`
- File selection panel → use `FileSelectionPanel`

Avoid:

- Copy-pasting UI code between tools
- Creating slightly different versions of the same widget
